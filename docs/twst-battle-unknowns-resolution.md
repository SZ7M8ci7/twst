# ツイステ バトル仕様 確定事項と不明点

作成日: 2026-04-27

この文書は、`twst-battle-emulator-spec.md` に対する補助資料です。ユーザー提供仕様で確定したこと、既存値から推定すること、まだ不明点として残すことを分けて記録します。

重要方針:

- `https://lyou-free.com/` に記載されている確率値は信頼度が低いため採用しない。
- ユーザー提供仕様で明示された値は、このエミュレーター仕様の確定値として扱う。
- 効果値で未指定のものは、既存の効果値から変化割合を推測して推定値を置く。
- やけどの割合/継続ターンと、各バトルの敵HP/火力/攻撃回数/バフ/行動パターンは都度入力する。
- 推定値は `estimated` として扱い、実測値が入った場合は上書きできる。

## 1. 確定事項

### ダメージ乱数と丸め

- ダメージ乱数は `0.95` 以上 `1.05` 以下の一様分布。
- 実ゲーム再現モードの丸めは小数点以下切り上げ。
- プロジェクト互換モードでは既存実装の `Math.round` や `Math.floor` を再現してよい。

### 暗闇

- 暗闇中の MISS 率は `21.6%`。
- MISS 判定は魔法単位。
- MISS 判定に入った場合、同じ魔法内の連撃はすべて MISS。
- 同ターン内に複数の暗闇効果が有効な場合、確率は加算する。

### 回避

- 回避(極小): `9.2%`
- 回避(小): `14.8%`
- 回避(中): `23.7%`
- 回避(大): `38.0%` 暫定推定
- 回避(極大): `61.0%` 暫定推定
- 判定単位はヒット単位。
- 同じ魔法の連撃でも、片方だけ外して片方だけ当たることがある。
- 複数回避バフは確率加算。

### 呪い

- 呪いは HP 回復と HP 継続回復を無効化する。
- 呪い(極小): `22.3%` 暫定推定
- 呪い(小): `33.4%` 暫定推定
- 呪い(中): `44.6%`
- 呪い(大): `66.9%` 暫定推定
- 呪い(極大): `100%`

### 凍結

- 既存バフは残る。
- 同じターンであっても、凍結後に発動したバフは発生しない。
- 凍結前に発動したバフは有効。
- 凍結(極小): `22.3%` 暫定推定
- 凍結(小): `33.4%` 暫定推定
- 凍結(中): `44.6%` 暫定推定
- 凍結(大): `66.9%` 暫定推定
- 凍結(極大): `100%`

### クリティカル

- クリティカル(極小): `10.0%` 暫定推定
- クリティカル(小): `19.1%`
- クリティカル(中): `33.3%`
- クリティカル(大): `50.0%`
- クリティカル(極大): `100.0%`
- 試験シミュレーターではヒット単位で発動判定を行い、発動したヒットだけ `1.25` 倍にする。

### デバフ解除

- 複数デバフがある場合はすべて解除する。
- デバフはすべて解除対象。
- 同じターンでは、解除より先に付与されたデバフは解除される。
- 解除後に付与されたデバフは有効。

### ガッツ

- 複数ガッツが付与された場合は、その回数分耐える。
- 毒は存在しない。
- やけどでは HP 0 以下にならないため、ガッツは発動しない。
- 複数対象は、その対象の数だけ付与され、その対象が攻撃された時に発動する。

### やけど

- ダメージ割合は効果ごとに都度設定する。
- ダメージ基準は対象カード本体 HP の割合。バディ HP は含めない。
- 発動タイミングは、そのターンのダメージ計算がすべて終わった後。
- 小数点以下は切り上げ。
- 継続ターンは効果ごとに都度設定する。
- やけどでは HP 0 以下にならない。

### 攻撃順とターゲット

- 攻撃は常に初手同士、後手同士で行われる。
- プレイヤー先攻ターンは `自分1 -> 敵1 -> 自分2 -> 敵2`。
- プレイヤー後攻ターンは `敵1 -> 自分1 -> 敵2 -> 自分2`。
- BS/BASIC と DF/DEFENCE は 1/3/5T をプレイヤー先攻、2/4T をプレイヤー後攻として扱う。
- ATK/ATTACK は最大10T。1T-5T は通常通り 1/3/5T をプレイヤー先攻、2/4T をプレイヤー後攻とし、6T-10T は後攻スタートとしてすべて `敵1 -> 自分1 -> 敵2 -> 自分2` で扱う。
- ターゲット不可状態は存在しない。
- 個別の戦闘不能状態は存在しない。
- `味方選択` は、プレイヤー側効果では同ターンで選んだ2つの魔法のカードを、敵側効果では同ターンで同時選択された2つの敵行動スロットを、重複なしで対象にする。
- `相手` は、初手魔法なら敵初手、後手魔法なら敵後手を対応対象にする1対象。
- `相手選択` は、そのターンで同時選択された相手側2枠を重複なしで対象にする。
- `味方全体` は味方側全員、`相手全体` は選択されていないカード/敵スロットも含めた相手側全員を対象にする。
- 敵からプレイヤーへ付与される暗闇、呪い、凍結、ダメージDOWN、やけどは、対応する初手/後手で攻撃されたカードを対象にする。別カードの魔法には波及しない。

### 試験ターン数と終了条件

- BS/BASIC と DF/DEFENCE は最大 5T。
- ATK/ATTACK は最大 10T。
- 最大ターン前でも、敵 HP または味方総 HP が 0 になった時点で終了する。
- 味方総 HP が 0 になった場合はリタイア扱いとし、スコアは `0`。

### プレイヤー手札

- 5枚デッキの選択済み魔法10個を1サイクルとして扱う。
- 1T目開始時に10個から5個を表示する。
- 毎ターン表示中の魔法から2個を選択し、消化後に隠れている未表示魔法から補充する。
- 4T以降は未表示魔法が少なくなり、残りの出現内容が確定していく。
- ATK/ATTACK で5T終了時に10個を消化した場合、6T目開始時に次サイクルを再抽選する。

### DUO シミュレーション

- 複数 DUO 条件は、エミュレーター上ではユーザーにターンごとの有効な組み合わせを最初に選択させる。
- DUO は同じターンのもう片方が DUO 相手キャラの場合のみ有効。
- 最初に決めたターンごとの有効な組み合わせで指定回数のシミュレーションを行う。
- 獲得できたスコアの分布をヒストグラムで表示する。
- エミュレーター側で DUO の追加制限や自動最適化は行わない。

### 敵行動

- 敵の火力、攻撃回数、バフ、HP は都度入力する。
- 各行動は条件に反しない限り独立してランダム。
- 5T バトルでは敵行動を合計 10 回分用意する。
- ATK/ATTACK 10T では、前半5Tの10行動構築と同じ組み合わせを後半6T以降にも採用しうる。

| 敵構成 | 10回分の作り方 |
| --- | --- |
| 3キャラで各3パターン | 9種類をすべて1回ずつ採用し、どれか1種類を追加で1回採用する。選択順はランダムで、最初に同じ行動が2回続くこともある。 |
| 3キャラで `3,3,4` の計10パターン | 10種類をそれぞれ1回ずつ採用する。 |
| 4キャラ以上 | 各キャラが最低1回は保持パターンのどれかを使う。残りは合計10回になるまで、未使用行動から選ぶ。 |
| 2キャラ | 各キャラが保持しているパターンをすべて採用する。合計10回に満たない分は重複して選ぶ。 |

すべてのパターンで同じ行動が3回以上選ばれることは存在しない。ただし、ATK試験の10Tではその組み合わせを後半の6T目以降でも採用するため、戦闘全体を通して同じ行動が4回行われることはあり得る。

### 係数差異

- ATKUP極大は `80%`。
- ダメージUP極大は `22.5%`。
- `回復&継続回復(中)` は存在しないため考慮不要。
- HP 回復は魔法行動時に即時発生し、ATK 基準で計算する。
- HP 継続回復はバフ付与とターン終了時の回復発生を分け、バディ HP を含まないカード本体 HP 基準で各ターン判定する。
- ダメージ UP/DOWN 系は既存シミュレータと同じく魔法倍率項に加減算する。戦闘中に付与されたダメージDOWN/被ダメージDOWNも最終倍率ではなく魔法倍率項から減算する。
- 凍結後に発動する継続回復バフは付与されない。呪いは即時回復・継続回復の各回復発生時点で回復量を `0` にする。
- バディ追加効果は `chara.json` 上の `ダメージUP`、`属性ダメージUP`、`クリティカル`、`回避`、`被ダメージDOWN`、`属性被ダメージDOWN`、状態異常無効、`HP継続回復(極小)` を対象にする。
- バディ追加効果は戦闘開始時に発動する `99T` 効果として扱う。敵の凍結では止まらず、呪いは回復発生時に阻害する。バフ解除は魔法で付与された効果だけに適用し、追加バディ効果は解除しない。
- バディ継続回復は毎ターン `ceil(カード本体HP * 10%)`。戦闘終了ターンは通常の継続回復と同じく発生しないため、合計回復量は終了タイミングで変わる。

## 3. 残る不明点

仕様として残る不明点はかなり少ない。現時点で残すのは以下。

- 動画内にしかない追加仕様。字幕は取得できなかったため、本文で明示された仕様以上は補わない。
- 新しい効果種別が追加された場合の処理。既存効果値から比率推定できる場合は推定し、推定できない種類は追加仕様待ちにする。
- 推定値の丸め桁。現ドキュメントでは効果値を小数第1位で丸める。
- やけどと HP 継続回復が同じターン終了処理で同時に発生した場合の厳密な前後順。現在の試験シミュレーターは、ユーザー提供の「やけどはそのターンのダメージ計算後」を優先して、やけど処理後に継続回復を処理する。

次のものは不明点ではなく、実行時入力データとして扱う。

- やけど効果ごとのダメージ割合。
- やけど効果ごとの継続ターン。
- 各バトルの敵 HP、火力、攻撃回数、バフ、行動パターン。
